# Proposal-Writing：从分散材料到完整课题申报活页

## Purpose

本 skill 是教学/申报 skill 流水线的**最终输出节点**。学生已经通过前序 skill 完成了：
- `topic-to-idea` → 研究设想（教学版/申报版）
- `idea-to-perspective` → 理论视角（教学版：读书报告；申报版：理论框架深化方案）
- `perspective-to-literature` → 批判性文献综述（教学版：2000-3000 字；申报版：1000-1200 字压缩版 + 前期成果）
- `literature-to-design` → 研究设计（教学版：6-8 周；申报版：3 年周期）

本 skill 将这些**分散的研究材料**整合为一份**完整的课题申报活页/开题报告**，严格遵循活页标准格式。

支持**双模式输出**：
- **教学版（teaching）**：逻辑一致、格式规范的课程作业/开题报告，强调概念对齐和学术规范
- **申报版（application）**：评审导向、竞争优化的国家社科基金活页，强调前期积累、匿名合规、字数精修、评审吸引力

**核心设计原则**：
- 教学版："拼接 + 改写 + 增补"——充分利用已有材料，确保逻辑一致性
- 申报版："评审导向 + 字数精修 + 匿名合规 + 吸引力优化"——每个章节都回答"评审专家会怎么看？"

## Mode Selection

用户未指定模式时，默认输出**教学版**。如果用户明确说"申报""课题""基金""立项""活页"，自动切换为**申报版**。

### 申报版输出格式

```
国家社会科学基金年度项目课题论证活页
课题名称：[从选题说明中提取并精炼]

1. [选题说明]
   ...

2. [选题依据]
   ...

3. [研究内容]
   ...

4. [创新之处]
   ...

5. [预期成果]
   ...

6. [参考文献]
   ...

说明：1.活页文字表述中不得出现任何可能透露申请人身份的信息。
      2.课题名称要与《申请书》一致，一般不加副标题。
```

**总字数约束**：不超过 7000 字（其中选题说明不超过 300 字）。

## Arguments

用户已提供：`$ARGUMENTS`

解析为文件路径列表或内容块：
1. **选题说明文件**（Markdown）：研究设想
2. **读书报告/理论框架文件**（Markdown）：理论视角
3. **文献综述文件**（Markdown）：批判性文献综述
4. **研究设计文件**（Markdown）：3.1 [研究内容] + 3.2 [创新之处]
5. **活页模板文件**（可选，.doc/.docx/.md）：如提供，读取其格式要求

**申报版额外需要**：
6. **前期成果列表**（3-5 项）：用于活页前期成果部分

## Workflow

### Step 0: 读取所有输入文件（双版本通用）

并行读取所有输入文件，提取以下关键信息：

| 章节 | 主要输入来源 | 提取内容 |
|------|------------|----------|
| 1. 选题说明 | 选题说明文件 | 研究题目、核心问题、研究视角、核心概念 |
| 2. 选题依据 | 文献综述文件（主体）+ 读书报告（理论价值） | 学术史梳理、研究动态、三个盲区、独到价值 |
| 3. 研究内容 | 研究设计文件（3.1） | 主要目标、重点难点、整体框架、研究计划、可行性 |
| 4. 创新之处 | 研究设计文件（3.2） | 学术观点创新、研究方法创新 |
| 5. 预期成果 | 新写（基于文献综述的应用价值） | 成果形式、宣传转化、学术价值、社会效益 |
| 6. 参考文献 | 文献综述文件（工作参考文献） | 中英文参考文献列表 |
| **申报版额外** | 前期成果列表 | 3-5 项前期相关成果 |

**一致性检查**：读取后立即检查四个文件中的核心概念是否一致：
- 选题说明中的"研究视角"与读书报告中的理论学者/著作是否匹配？
- 选题说明中的"核心概念"与文献综述中的关键概念是否一致？
- 文献综述中的假设与研究设计中的目标是否对应？

如果不一致，优先以**文献综述**中的表述为准（因为它是整合后的最终版本），在拼接时对不一致的章节进行改写。

### 【申报版】Step 1-6：撰写六章（评审导向优化）

#### Step 1: 撰写 1. [选题说明]（≤300 字，评审第一眼）

**策略**：直接采用选题说明文件的主体内容，但进行**概念对齐改写**和**评审吸引力优化**。

**改写规则**：
- 如果选题说明中的理论视角/核心概念与文献综述不一致，以文献综述为准进行微调
- 确保 300 字以内包含：具体问题、研究视角、核心概念
- **申报版额外**：第五句必须包含"理论价值 + 政策价值"双重贡献

**申报版选题说明检查清单**：
- [ ] 第一句是否有具体数据和痛点？
- [ ] 第二句是否清晰提出研究问题（以问号或"本研究试图回答"开头）？
- [ ] 第三句是否即时锚定核心概念？
- [ ] 第四句是否让核心概念参与机制叙述？
- [ ] 第五句是否包含双重价值（理论+政策）？
- [ ] 总字数 ≤ 300 字？
- [ ] 无"笔者""我的"等身份暴露词？

---

#### Step 2: 撰写 2. [选题依据]（2000-2500 字，评审核心）

**策略**：学术史梳理直接采用文献综述正文的**批判性梳理部分**，研究进展和独到价值从文献综述末尾的"解释缺口"和理论嫁接中提取。

**申报版与教学版的区别**：

| 维度 | 教学版 | 申报版 |
|:-----|:-------|:-------|
| 文献梳理篇幅 | 可详细展开（1500 字） | 压缩至 1000-1200 字，保留骨架 |
| 独到价值 | 写理论贡献即可 | 必须写"理论价值 + 应用价值" |
| 前期成果 | 不呈现 | **必须呈现** 3-5 项前期相关成果 |
| 文献对话 | 展示批判性 | 展示批判性 + **点名具体对手** |

**申报版结构**：
```
一、国内外相关研究的学术史梳理及研究进展
    （一）英文文献：[路径1] + [路径2] + [关键发现] + [盲区]
    （二）中文文献：[实证研究] + [理论分析] + [关键发现] + [盲区]
    （三）从[理论视角]重读现有文献：[三个盲区]

二、相对于已有研究的独到学术价值和应用价值
    学术价值：[理论空白 + 概念贡献 + 与具体文献的对话]
    应用价值：[政策诊断工具 + 可操作路径 + 预期社会效益]

三、前期相关研究成果
    [3-5 项，匿名处理，按相关性排序]
```

**前期成果呈现规则**：
- 不署申请人姓名
- 合著者只列姓氏或写"某作者等"
- 每项成果附一句话说明与选题的关系
- 如果年份-期刊组合过于罕见，建议增加 1-2 篇其他期刊论文稀释

---

#### Step 3: 撰写 3. [研究内容]

**策略**：直接采用研究设计文件中的 **3.1 [研究内容]** 部分，申报版几乎不做修改（因为 literature-to-design 申报版已经按 NSSFC 标准输出）。

**申报版检查项**：
- [ ] 主要目标是否明确（3-4 个递进目标）
- [ ] 重点难点是否分开列出，且难点有具体解决方案
- [ ] 整体框架是否以树形目录/章节提纲呈现
- [ ] 研究计划是否包含：数据来源、样本策略、变量测量、分析方法、时间线
- [ ] 可行性论证是否具体说明"为什么申请人能做"（数据可及、方法可控、团队支撑、单位支持）
- [ ] 是否有自然实验或准实验设计？
- [ ] 时间线是否为 3 年（36 个月）？

---

#### Step 4: 撰写 4. [创新之处]

**策略**：直接采用研究设计文件中的 **3.2 [创新之处]** 部分，申报版几乎不做修改。

**申报版检查项**：
- [ ] 学术观点创新是否包含：理论视角独特性、核心概念创新、与现有文献的对话
- [ ] 研究方法创新是否包含：因果识别策略、数据组合创新、可复制性
- [ ] 创新点是否 ≤ 3 条（每维度 1-2 条）？
- [ ] 每条创新是否有具体的理论对手（如"与 Tang 2020 不同……"）？
- [ ] 无"填补了空白""首次研究"等空话？

---

#### Step 5: 撰写 5. [预期成果]

**策略**：基于文献综述末尾的"应用价值"部分**新写**，保持简洁（略写）。

**申报版必含要素**（每项 1-2 句话）：
- 成果形式（论文、报告、专著等）
- 宣传转化（向谁提交、如何推广）
- 预期学术价值（理论贡献）
- 预期社会效益（如果经验证据支持假设，对政策有何启示）
- **阶段性成果时间节点**（第 1 年、第 2 年、第 3 年分别完成什么）

**字数控制**：200-400 字。

---

#### Step 6: 撰写 6. [参考文献]

**策略**：直接采用文献综述文件中的"工作参考文献"部分，格式化为活页要求的样式。

**申报版格式规则**：
- 英文文献在前，中文文献在后
- 每条参考文献单独一行，不使用项目符号（`-` `*` `+`）
- 遵循作者-日期制：Author. Year. "Title." *Journal* Volume(Issue): Pages.
- 中文文献格式：作者. 年份. "标题." *期刊名* (期): 页码.
- **申报版额外**：总量建议 20-25 条，近五年文献占比 ≥ 60%，CSSCI/SSCI 占比 ≥ 70%

---

### 【申报版】Step 7: 全局优化（字数精修 + 匿名合规 + 评审测试）

申报版在 Step 6 之后，必须执行三轮优化：

#### 7.1 字数精修

| 章节 | 建议字数 | 当前字数 | 状态 | 压缩策略 |
|:-----|:---------|:---------|:-----|:---------|
| 1. 选题说明 | ≤300 | [统计] | [✅/⚠️] | 删减修饰语，保留"主体+困境+数据" |
| 2. 选题依据 | 2000-2500 | [统计] | [✅/⚠️] | 删减文献细节论证，保留核心论点 |
| 3. 研究内容 | 1500-2000 | [统计] | [✅/⚠️] | 删减技术操作步骤，保留框架逻辑 |
| 4. 创新之处 | 600-800 | [统计] | [✅/⚠️] | **绝不压缩** |
| 5. 预期成果 | 200-400 | [统计] | [✅/⚠️] | 保持简洁 |
| 6. 参考文献 | 400-600 | [统计] | [✅/⚠️] | 不计入主体字数 |
| **总计** | **≤7000** | **[统计]** | **[✅/⚠️]** | — |

**如果字数超限**：
1. 优先压缩"选题依据"中的文献梳理部分（保留核心论点，删减细节论证）
2. 其次压缩"研究内容"中的技术细节（保持框架和逻辑，删减具体操作步骤）
3. 绝不压缩"创新之处"（这是评审的核心关注点）

#### 7.2 匿名合规性检查（20 项）

加载 `references/nssfc-anonymity-checklist.md` 执行扫描。

**高风险项（必须修改）**：
- [ ] 无"笔者""我的研究""我发现"等第一人称
- [ ] 无学校/院系名称
- [ ] 无导师/合作者姓名
- [ ] 前期成果署名已匿名处理

**中风险项（建议修改）**：
- [ ] 无过于具体的地名（区/街道级）
- [ ] 无只有本单位才有的数据库名称
- [ ] 无个人 GitHub/知乎/公众号等账号
- [ ] 前期成果年份-期刊组合不过于罕见

**低风险项（可选修改）**：
- [ ] 无致谢语
- [ ] 无过于个性化的写作风格

#### 7.3 评审视角 3 分钟测试

模拟评审专家的阅读过程，检查活页在 3 分钟内能否传递核心信息：

**第 1 分钟（选题说明）**：
- [ ] 30 秒内能否说清研究问题？
- [ ] 问题是否具有现实紧迫性和理论张力？

**第 2 分钟（选题依据 + 创新之处）**：
- [ ] 能否感受到文献缺口的真实性？
- [ ] 能否记住 1-2 个核心创新点？
- [ ] 创新点是否有具体的理论对手？

**第 3 分钟（研究内容框架 + 可行性）**：
- [ ] 整体框架是否清晰、逻辑是否自洽？
- [ ] 数据来源是否权威、方法是否可行？
- [ ] 申请人是否"够格"做这个项目？

**如果任一检查未通过**，返回相应章节进行针对性修改。

## 常见错误与修复

| 错误 | 表现 | 修复 |
|:-----|:-----|:-----|
| 字数超限 | 选题依据写得过于详细 | 删减英文/中文文献的细节论证，保留核心论点 |
| 概念不一致 | 选题说明用"去空间化"，文献综述用"翻译阻断" | 以文献综述为准统一概念，改写选题说明 |
| 泄露身份 | 出现"我的导师""笔者在浙江大学" | 全局替换为"本研究"，删除学校/人名 |
| 重复论证 | 选题依据和研究内容都详细写了文献梳理 | 选题依据聚焦"批判性梳理"，研究内容聚焦"研究执行" |
| 假设编号混乱 | 文献综述说 H1，研究内容说 H2 | 统一为文献综述中的编号 |
| 遗漏预期成果 | 只有 5 个章节 | 补充第 5 章，保持简洁 |
| 参考文献格式错误 | 使用 Markdown 列表（`- ` `* `） | 改为段落分隔，每条一行 |
| 研究内容缺框架 | 没有列出章节提纲/目录 | 补充树形目录或章节提纲 |
| **申报版**：前期成果缺失 | 活页中没有前期成果部分 | 整合 3-5 项前期相关成果 |
| **申报版**：前期成果暴露身份 | 署名未匿名，年份-期刊组合罕见 | 匿名处理，增加其他期刊论文稀释 |
| **申报版**：创新之处过虚 | "填补了空白""首次研究" | 改为"与 [某文献] 不同，本研究……" |
| **申报版**：无自然实验 | 纯横截面描述，无因果识别 | 在研究设计中补充自然实验/准实验 |
