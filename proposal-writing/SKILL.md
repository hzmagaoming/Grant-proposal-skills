# Proposal-Writing：从分散材料到完整课题申报活页

## Purpose

本 skill 是教学 skill 流水线的**最终输出节点**。学生已经通过前序 skill 完成了：
- `topic-to-idea` → 300字选题说明
- `idea-to-perspective` → 读书报告（理论视角）
- `perspective-to-literature` → 批判性文献综述（含假设）
- `literature-to-design` → 研究设计（3.1+3.2格式）

本 skill 将这些**分散的研究材料**整合为一份**完整的课题申报活页/开题报告**，严格遵循活页标准格式。

**核心设计原则**：不是从头撰写，而是**"拼接 + 改写 + 增补"**——充分利用已有材料，同时确保活页各章节之间的逻辑一致性。

---

## 输出格式（严格遵循）

本 skill 的输出格式严格匹配国家社会科学基金课题论证活页标准：

```
国家社会科学基金年度项目课题论证活页
课题名称：[从选题说明中提取并精炼]

1. [选题说明]
   选题所研究的具体问题、研究视角和核心概念（300字以内）。

2. [选题依据]
   国内外相关研究的学术史梳理及研究进展（略写）；
   相对于已有研究特别是国家社科基金同类项目的独到学术价值和应用价值。

3. [研究内容]
   课题研究的主要目标、重点难点、整体框架、研究计划及其可行性等。
   （框架思路要列出提纲或目录）

4. [创新之处]
   在学术观点、研究方法等方面的特色和创新。

5. [预期成果]
   成果形式、宣传转化及预期学术价值和社会效益等。（略写）

6. [参考文献]
   开展本课题研究的主要中外参考文献。（略写）

说明：1.活页文字表述中不得出现任何可能透露申请人身份的信息。
      2.课题名称要与《申请书》一致，一般不加副标题。
```

**总字数约束**：不超过7000字。

---

## Arguments

用户已提供：`$ARGUMENTS`

解析为文件路径列表（至少4个，最多5个）：
1. **选题说明文件**（Markdown）：300字研究设想，含研究题目、问题、视角、概念
2. **读书报告文件**（Markdown）：500-800字，含理论嫁接、批判回应、视角改写
3. **文献综述文件**（Markdown）：批判性文献综述，含中英文文献地图、解释缺口、假设
4. **研究设计文件**（Markdown）：3.1 [研究内容] + 3.2 [创新之处]
5. **活页模板文件**（可选，.doc/.docx/.md）：如提供，读取其格式要求；如不提供，使用默认格式

---

## Workflow

### Step 0: 读取所有输入文件

并行读取所有输入文件，提取以下关键信息：

| 章节 | 主要输入来源 | 提取内容 |
|------|------------|----------|
| 1. 选题说明 | 选题说明文件 | 研究题目、核心问题、研究视角、核心概念 |
| 2. 选题依据 | 文献综述文件（主体）+ 读书报告（理论价值） | 学术史梳理、研究动态、三个盲区、独到价值 |
| 3. 研究内容 | 研究设计文件（3.1） | 主要目标、重点难点、整体框架、研究计划、可行性 |
| 4. 创新之处 | 研究设计文件（3.2） | 学术观点创新、研究方法创新 |
| 5. 预期成果 | 新写（基于文献综述的应用价值） | 成果形式、宣传转化、学术价值、社会效益 |
| 6. 参考文献 | 文献综述文件（工作参考文献） | 中英文参考文献列表 |

**一致性检查**：读取后立即检查四个文件中的核心概念是否一致：
- 选题说明中的"研究视角"与读书报告中的理论学者/著作是否匹配？
- 选题说明中的"核心概念"与文献综述中的关键概念是否一致？
- 文献综述中的假设与研究设计中的目标是否对应？

如果不一致，优先以**文献综述**中的表述为准（因为它是整合后的最终版本），在拼接时对不一致的章节进行改写。

---

### Step 1: 撰写 1. [选题说明]

**策略**：直接采用选题说明文件的主体内容，但进行**概念对齐改写**。

**改写规则**：
- 如果选题说明中的理论视角/核心概念与文献综述不一致，以文献综述为准进行微调
- 确保300字以内包含：具体问题、研究视角、核心概念
- 如果选题说明使用了与后续章节不同的理论术语（如"去空间化"vs"翻译阻断"），统一为文献综述中最终确立的概念

**示例改写**：
> 原文："引入'去空间化'概念——数字平台通过虚拟地址与数据壁垒将经营抽离物理空间"
> 改写："引入行动者网络理论（ANT）的'翻译'框架——将平台推荐算法重新概念化为具有自身利益的'非人类行动者'，追问为何'食品安全'无法被翻译为算法的运营规则"

---

### Step 2: 撰写 2. [选题依据]

**策略**：学术史梳理直接采用文献综述正文的**批判性梳理部分**（非文献地图表格），研究进展和独到价值从文献综述末尾的"解释缺口"和读书报告的理论嫁接中提取。

**结构**：
```
一、国内外相关研究的学术史梳理及研究进展
    （一）英文文献：[路径1] + [路径2] + [关键发现]
    （二）中文文献：[实证研究] + [理论分析] + [关键发现]
    （三）从ANT视角重读现有文献：[三个盲区]

二、相对于已有研究的独到学术价值和应用价值
    学术价值：[理论空白 + 概念贡献]
    应用价值：[政策诊断工具 + 可操作路径]
```

**拼接规则**：
- 英文文献部分：采用文献综述中"英文文献对平台监管失灵的分析主要沿着两条路径展开..."段落
- 中文文献部分：采用文献综述中"中文文献对幽灵外卖监管的研究呈现出与英文文献不同的关注焦点..."段落
- 三个盲区：直接采用文献综述末尾的总结段落
- 学术价值：从文献综述末尾的"基于上述批判性综述"段落中提取假设的学术意义
- 应用价值：从文献综述中淮北市检察院案例和政策建议中提取

**删减规则**：
- 删除文献综述中的"搜索摘要"、"文献地图表格"、"方法论图景"等元信息
- 删除"竞争性假设与选择"表格（活页不需要展示未选假设）
- 保留核心的批判性论证逻辑

---

### Step 3: 撰写 3. [研究内容]

**策略**：直接采用研究设计文件中的 **3.1 [研究内容]** 部分，几乎不做修改。

**检查项**：
- [ ] 主要目标是否明确（2-4个递进目标）
- [ ] 重点难点是否分开列出
- [ ] 整体框架是否以树形目录/章节提纲呈现
- [ ] 研究计划是否包含：数据来源、样本策略、变量测量、分析方法、时间线
- [ ] 可行性论证是否具体说明"为什么本科生/研究者能做"

---

### Step 4: 撰写 4. [创新之处]

**策略**：直接采用研究设计文件中的 **3.2 [创新之处]** 部分，几乎不做修改。

**检查项**：
- [ ] 学术观点创新是否包含：理论视角独特性、核心概念创新、与现有文献的对话
- [ ] 研究方法创新是否包含：数据采集独特性、自然实验锚点、可复制性

---

### Step 5: 撰写 5. [预期成果]

**策略**：基于文献综述末尾的"应用价值"部分**新写**，保持简洁（略写）。

**必含要素**（每项1-2句话）：
- 成果形式（论文、报告、代码等）
- 宣传转化（向谁提交、如何推广）
- 预期学术价值（理论贡献）
- 预期社会效益（如果经验证据支持假设，对政策有何启示）

**字数控制**：200-400字。

---

### Step 6: 撰写 6. [参考文献]

**策略**：直接采用文献综述文件中的"工作参考文献"部分，格式化为活页要求的样式。

**格式规则**：
- 英文文献在前，中文文献在后
- 每条参考文献单独一行，不使用项目符号（`-` `*` `+`）
- 遵循作者-日期制：Author. Year. "Title." *Journal* Volume(Issue): Pages.
- 中文文献格式：作者. 年份. "标题." *期刊名* (期): 页码.

---

### Step 7: 全局一致性检查与字数控制

**一致性检查清单**：
- [ ] 课题名称在活页开头、选题说明、研究内容中保持一致
- [ ] 核心概念（如"算法锁定""翻译阻断"）在各章节中表述一致
- [ ] 理论视角（如"行动者网络理论""Callon翻译框架"）在各章节中表述一致
- [ ] 假设编号（H1）在研究内容中提到时，与文献综述中的假设一致
- [ ] 时间、金额等数字在各章节中一致（如"2026年4月35.97亿罚单"）

**字数控制**：

| 章节 | 建议字数 | 弹性范围 |
|------|----------|----------|
| 1. 选题说明 | 250-300 | ≤300（硬性上限） |
| 2. 选题依据 | 2000-2500 | 可压缩至1800 |
| 3. 研究内容 | 1500-2000 | 可压缩至1200 |
| 4. 创新之处 | 600-800 | 可压缩至500 |
| 5. 预期成果 | 200-400 | 可压缩至200 |
| 6. 参考文献 | 400-600 | 不计入主体字数 |
| **总计** | **≤7000** | **硬性上限** |

**如果字数超限**：
1. 优先压缩"选题依据"中的文献梳理部分（保留核心论点，删减细节论证）
2. 其次压缩"研究内容"中的技术细节（保持框架和逻辑，删减具体操作步骤）
3. 绝不压缩"创新之处"（这是评审的核心关注点）

---

### Step 8: 去身份化处理

**活页硬性要求**：不得出现任何可能透露申请人身份的信息。

**检查并删除**：
- [ ] 学生姓名、学号、导师姓名
- [ ] 学校名称、院系名称
- [ ] 个人联系方式、邮箱
- [ ] "我的研究""笔者"等第一人称表述，改为"本研究"
- [ ] 过于个人化的经验描述（如"我在某平台实习时发现"）

**替换规则**：
- "我的选题" → "本研究"
- "笔者发现" → "研究发现"
- "我在读书报告中提出" → "本研究提出"
- "学生所在城市" → "某城市"（如果涉及具体城市名）

---

### Step 9: 保存输出

**文件名**：`开题报告_活页-[topic-slug]-[YYYY-MM-DD].md`

**文件内容**：完整活页文本，包含所有6个章节。

**辅助文件**（可选）：
- `活页_字数统计-[YYYY-MM-DD].md`：各章节字数明细

---

## 质量检查清单

完成输出前，逐一验证：

- [ ] 输出包含完整的6个章节（1-6），不增不减
- [ ] 1. 选题说明 ≤300字，包含问题、视角、概念
- [ ] 2. 选题依据包含：英文文献梳理、中文文献梳理、三个盲区、学术价值、应用价值
- [ ] 3. 研究内容包含：主要目标、重点难点、整体框架（提纲/目录）、研究计划、可行性
- [ ] 4. 创新之处包含：学术观点创新、研究方法创新
- [ ] 5. 预期成果包含：成果形式、宣传转化、学术价值、社会效益（略写）
- [ ] 6. 参考文献格式正确，无项目符号
- [ ] 总字数 ≤7000字
- [ ] 无申请人身份信息（姓名、学校、院系等）
- [ ] 核心概念在各章节中表述一致
- [ ] 课题名称在全文中保持一致
- [ ] 时间、金额等数字在各章节中一致
- [ ] 假设编号与文献综述一致
- [ ] 无"[CITATION NEEDED]"等占位符
- [ ] 无Markdown代码块、表格等格式标记残留在正文中

## 常见错误与修复

| 错误 | 表现 | 修复 |
|------|------|------|
| 字数超限 | 选题依据写得过于详细 | 删减英文/中文文献的细节论证，保留核心论点 |
| 概念不一致 | 选题说明用"去空间化"，文献综述用"翻译阻断" | 以文献综述为准统一概念，改写选题说明 |
| 泄露身份 | 出现"我的导师""笔者在浙江大学" | 全局替换为"本研究"，删除学校/人名 |
| 重复论证 | 选题依据和研究内容都详细写了文献梳理 | 选题依据聚焦"批判性梳理"，研究内容聚焦"研究执行" |
| 假设编号混乱 | 文献综述说H1，研究内容说H2 | 统一为文献综述中的编号 |
| 遗漏预期成果 | 只有5个章节 | 补充第5章，保持简洁 |
| 参考文献格式错误 | 使用Markdown列表（`- ` `* `） | 改为段落分隔，每条一行 |
| 研究内容缺框架 | 没有列出章节提纲/目录 | 补充树形目录或章节提纲 |
