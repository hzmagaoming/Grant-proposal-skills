# Topic-to-Idea Skill 制作说明

> 从模糊的政策议题到 300 字研究设想的自动化生成工具

---

## 一、起源

原始需求来自国家社会科学基金课题申报场景。研究者面对官方发布的几十项政策议题（如"数智社会治理""建设生育友好型社会"），不知道如何把宏大的政策表述转化为**具体的、可操作的、有理论深度的**研究问题。

初始版本是一个 4 步转化框架（诊断→定位→聚焦→检验），但输出不稳定、格式不固定、理论视角经常悬空。经过多轮迭代，最终收敛为**固定五句结构**。

---

## 二、核心设计：五句叙事结构

不是简单堆砌要素，而是强迫叙事咬合：

| 句序 | 功能 | 衔接机制 |
|:-----|:-----|:---------|
| 第一句 | 现实痛点 + 关键数据 | 用具体经验现象制造问题意识 |
| 第二句 | 制度悖论 | **必须用因果承接词**（"问题的根源在于"）与第一句咬合 |
| 第三句 | 独特视角 + 核心概念 | **破折号即时锚定**概念，避免悬空 |
| 第四句 | 预期震撼发现 | **概念必须参与机制叙述**，形成咬合 |
| 第五句 | 研究意义 | 用"打破…悖论""构建…框架"替代平淡的"提供理论支撑" |

**关键洞见**：第三、四句的"概念咬合"是整段话的灵魂。很多初稿的失败在于：抛出理论概念后，第四句不再提及它，导致视角和发现割裂。

---

## 三、改造过程的关键节点

1. **议题来源固定化**  
   从开放式输入改为必须从官方议题库中选择，消除模型自由发挥的漂移。

2. **输出格式一段化**  
   从分点列表改为一段连续文本，模拟真实学术提案的语感。

3. **措辞硬度规则**  
   明确禁止"削弱""影响""不足"等疲软动词，改用"制造""消解""挤压""强制""滑向"。

4. **参考文献联动**  
   将国家社科基金课题论证活页模板和 2026 年重点研究项目文件放入 references，使 skill 输出可对照官方格式标准。


仓库托管于 GitHub：[hzmagaoming/Grant-proposal-skills](https://github.com/hzmagaoming/Grant-proposal-skills)

---

## 五、使用方式

在 Kimi CLI 中，直接说：

> "以'人工智能发展和治理'为议题生成一个研究设想"

Kimi 会自动调用该 skill，输出一段 300 字以内的五句研究设想。

---

## 六、局限性

- 议题库目前固定为国家社科基金 2026 年重点方向，如需扩展需手动编辑 SKILL.md
- 对于跨学科交叉议题，模型可能倾向于选择单一学科视角，需人工微调
- 第五句（研究意义）在字数紧张时容易被压缩过度

---

*本 skill 由人工设计框架 + 多轮对话迭代优化而成，核心创新在于"五句咬合结构"对学术写作连贯性的强制约束。*
