# Perspective-to-Literature：从理论视角到批判性文献综述

## Purpose

本 skill 是 `idea-to-perspective` 的下游配套工具。学生已经完成了读书报告、选定了理论视角；现在需要：
1. 系统搜索中英文文献（中文占比不低于 40%）
2. 在文献地图基础上提出假设
3. 撰写一篇批判性文献综述——不是罗列文献，而是展示对话、张力与盲区

支持**双模式输出**：
- **教学版（teaching）**：提出 3 个真正竞争的假设供学生选择，撰写 2000–3000 字完整批判性综述，培养文献批判能力
- **申报版（application）**：提出 1 个主假设 + 2 个备用假设（用于稳健性检验），撰写 1000–1200 字活页压缩版综述，整合前期成果列表，直接服务于国家社科基金/课题申报

## Mode Selection

用户未指定模式时，默认输出**教学版**。如果用户明确说"申报""课题""基金""立项""活页"，自动切换为**申报版**。

---

## 上游与下游

- **上游输入**：`idea-to-perspective` 的输出（含已选学者/著作/理论视角的读书报告或选题说明）
- **下游衔接**：选定假设后，可进入 `literature-to-design`（设计研究方案）

---

## 核心逻辑

文献综述不是"我读了多少篇文献"的清单，而是一个**论证**：
1. **现有文献在说什么** → 中英文文献的核心发现与共识
2. **现有文献的盲区在哪里** → 中英文文献各自和共同的缺口
3. **用我的理论视角重读文献能看到什么** → 理论透镜揭示的新问题
4. **因此我应该检验什么假设** → 从缺口到假设的推导链
5. **批判性综述** → 将上述论证写成一篇有对话、有张力、有盲区的文章

### Phase 3: 假设提案（教学版 vs 申报版）

#### 【教学版】Step 7: 提出 3 个竞争性假设

（保持原有设计不变：3 个真正竞争的假设，用户选择后进入 Phase 4。详见原 SKILL.md。）

#### 【申报版】Step 7: 提出 1 个主假设 + 2 个备用假设

基于文献地图和缺口，提出 **1 个主假设**（核心研究假设）和 **2 个备用假设**（用于稳健性检验或扩展分析）。

**主假设（H1）必须**：
- 直接回应核心缺口
- 使用学生已选的理论视角
- 有明确的数据可行性
- 能在 3 年内完成检验

**备用假设（H2/H3）必须**：
- 与 H1 形成逻辑递进或边界检验关系
- H2 通常是 H1 的**机制检验**（中介/调节）
- H3 通常是 H1 的**稳健性检验**（不同情境/不同数据/不同方法）
- **不是与 H1 竞争的假设**，而是 H1 的"护城河"

**每个假设必须标注**：

| 维度 | 标注内容 |
|------|----------|
| **理论深度** | 浅（直接应用已有理论）/ 中（扩展理论边界）/ 深（修正或挑战理论） |
| **数据可行性** | 高（公开数据/二手数据可做）/ 中（需收集一手数据）/ 低（需实验/长期田野） |
| **方法复杂度** | 低（描述性/回归）/ 中（工具变量/匹配/固定效应）/ 高（实验/DiD/RD/混合方法） |
| **在活页中的位置** | 选题依据末尾 / 研究内容第三章 |

**申报版输出格式**：

```
【假设体系】

基于上述文献地图，我们从 [缺口描述] 出发，构建以下假设体系。
三个假设形成"主效应→机制检验→稳健性检验"的递进关系，
而非相互竞争关系。

---

【主假设 H1：[简称]】
理论深度：[深/中/浅] | 数据可行性：[高/中/低] | 方法复杂度：[低/中/高]

H1: [正式假设陈述]

推导逻辑：
[从缺口到假设的推导，2-3句话]

与文献的对话：
- 直接回应 [某文献] 的 [某盲区]
- 挑战 [某主流主张] 的默认假设

在活页中的位置：选题依据末尾（从缺口到假设的推导段）

---

【备用假设 H2：机制检验】
理论深度：[中] | 数据可行性：[中] | 方法复杂度：[中]

H2: [机制假设陈述，如"清晰化逻辑通过消解自由裁量空间中介了标准化程序对调解质量的影响"]

功能：检验 H1 的因果机制，排除"虚假相关"解释

在活页中的位置：研究内容第五章（机制分析章节）

---

【备用假设 H3：稳健性检验】
理论深度：[浅] | 数据可行性：[高] | 方法复杂度：[低]

H3: [稳健性假设陈述，如"在网格化推进较晚的地区，清晰化逻辑对调解质量的影响较弱"]

功能：检验 H1 的情境边界，增强结论的普适性

在活页中的位置：研究内容第五章（子样本比较或异质性分析）
```

**重要规则**：
- 3 个假设不是竞争关系，而是"主效应→机制→稳健性"的递进关系
- 申报版不要求用户选择假设（不像教学版的"3 选 1"），而是将三个假设作为一个体系呈现
- 如果评审专家质疑"为什么不多做几个竞争性假设"，解释："本研究聚焦于检验 [理论视角] 的核心预测，竞争性假设的检验留给后续研究"

### Phase 4: 批判性文献综述撰写

#### 【教学版】Step 9-10: 撰写 2000–3000 字完整综述

（保持原有设计不变。详见原 SKILL.md。此处省略以节省上下文，执行时完整加载。）

#### 【申报版】Step 9-10: 撰写 1000–1200 字活页压缩版综述

申报版综述不是教学版综述的简单压缩，而是**重新组织论证逻辑**，以适应活页评审场景：

**结构变化**：

| 段落 | 字数 | 功能 | 与教学版的区别 |
|:-----|:-----|:-----|:---------------|
| §1 引言（100-150字） | 更短 | 直接抛出 puzzle + 缺口 | 删减背景铺陈 |
| §2 英文文献（250-300字） | 压缩 | 核心主张→内部挑战→盲区 | 删减案例细节，保留论点骨架 |
| §3 中文文献（250-300字） | 压缩 | 独特发现→局限→与英文对话 | 同上 |
| §4 理论对话（300-400字） | 保持 | 核心段落：用已选视角重读文献 | 保持核心地位，不减压 |
| §5 批判性综合（150-200字） | 压缩 | 三个盲区→为什么需要新研究 | 更简洁 |
| §6 从缺口到假设（100-150字） | 压缩 | 导向 H1 | 直接呈现 H1，不讨论 H2/H3 |

**压缩原则**：
1. **保留论证骨架，删减肌肉**：保留"A 主张 X，但 B 质疑，因为 Y"的批判性结构，删减具体案例和数据来源描述
2. **保留理论对话，删减文献细节**：§4 是评审专家最关注的部分，保持 300-400 字；§2-3 压缩至 250-300 字
3. **点名具体文献，但不展开**："Tang (2020) 主张……但 Read (2012) 质疑……"（两句话解决一篇文献）

**零 bullet list 规则**：申报版同样不允许 bullet list，全文为段落 prose。

### Phase 5: 前期成果整合（仅申报版）

#### 【申报版】Step 11: 整合前期成果

在活页压缩版综述之后，生成**前期相关研究成果列表**。

**格式要求**（严格遵循活页规定）：
```
【前期相关研究成果】

1. [成果形式，如论文/专著/报告]：[标题]（匿名处理，不署申请人姓名）
   发表刊物/出版社：[期刊名/出版社名]
   发表时间：[年份]
   与选题的关系：[一句话说明该成果如何为本研究奠定基础]

2. ...

3. ...

（共列出 3-5 项，按与选题的相关性排序）
```

**匿名处理规则**：
- 不署申请人姓名
- 如果成果是合著，只列出所有作者姓氏（如"张三、李四"改为"Zhang, Li"或"某作者等"）
- 如果成果年份+期刊组合过于罕见（如某年某期刊仅发表了 1 篇该主题论文），建议增加 1-2 篇其他期刊论文稀释识别度

**前期成果与选题的相关性说明模板**：
- "该文首次提出'XX'概念，为本研究的理论框架奠定基础。"
- "该文基于 XX 数据，揭示了 XX 机制，为本研究的经验分析提供方法参考。"
- "该文的文献梳理涵盖了 XX 领域，为本研究的缺口识别提供起点。"

## 质量检查清单

### 教学版专用
- [ ] 提出了 3 个真正竞争的假设（不是互补关系）
- [ ] 用户已明确选择其中一个假设
- [ ] 综述总字数 2000–3000 字
- [ ] 全文无 bullet list（除结构大纲外）

### 申报版专用
- [ ] 主假设 H1 直接回应核心缺口，有明确数据可行性
- [ ] 备用假设 H2/H3 与 H1 形成递进关系，而非竞争关系
- [ ] 综述压缩版 1000–1200 字，六段结构，零 bullet list
- [ ] 前期成果 3-5 项，与选题直接相关，已匿名处理
- [ ] 前期成果的年份-期刊组合不会罕见到可识别身份
- [ ] 保留文献中 CSSCI/SSCI 占比 ≥ 70%
- [ ] §4 理论对话段落字数 ≥ 300 字（评审最关注部分，不减压）
- [ ] 假设编号在各章节中一致（H1/H2/H3）

### 双版本通用
- [ ] 中文查询数量 ≥ 英文查询数量的 60%
- [ ] 至少 1 处中英文文献直接对话
- [ ] 中文文献占比 ≥ 40%（按引用数量计）
- [ ] 所有引用文献都来自搜索日志（无编造引用）
- [ ] §1 的 puzzle 在§5 中得到回应
- [ ] §6 的假设可从§5 的盲区直接推导

## 常见错误与修复

| 错误 | 表现 | 修复 |
|:-----|:-----|:-----|
| 教学版：中文文献沦为点缀 | 中文文献只在§3出现 | 强制中英文搜索并行；§2也应引用中文文献 |
| 教学版：假设互补而非竞争 | H1做主效应，H2做中介 | 三个假设必须对同一现象做出不同预测 |
| 申报版：假设过多且分散 | 提出 5-6 个假设，无主线 | 聚焦 1 个主假设 + 2 个备用假设，形成递进体系 |
| 申报版：综述压缩过度 | §4 理论对话被压到 <200 字 | §4 是评审最关注部分，保持 ≥300 字 |
| 申报版：前期成果凑数 | 列出与选题无关的成果 | 删除无关成果，宁缺毋滥 |
| 申报版：前期成果暴露身份 | 年份+期刊组合过于罕见 | 增加 1-2 篇其他期刊论文稀释识别度 |
| 申报版：缺口太泛 | "这个主题研究很少" | 必须点名具体文献、具体盲区、具体未检验假设 |
| 申报版：假设突然跳出来 | §6的假设与前文无逻辑连接 | 必须能从§5的盲区直接推导，写出推导句 |
