# Topic-to-Idea：从政策议题到研究设想

从给定的政策/社会议题库中选择一个话题，生成一段 300 字以内的研究设想。

## 输入：候选议题库

**必须从以下议题中选择一个作为输入 topic**（不得自行添加或改写）：

推进国家治理体系和治理能力现代化，发展全过程人民民主，建设社会主义法治国家，数字法治，涉外法治，繁荣发展新时代中国特色社会主义文化，建设文化强国，人文经济学，新大众文艺，健全公共文化服务体系，健全社会治理体系，数智社会治理，人工智能发展和治理，科技伦理，数智技术对社会结构、劳动就业、生活方式、人口行为等方面的影响，建设教育强国、科技强国、人才强国，健全与人口变化相适应的教育资源配置机制，建设健康中国，建设体育强国，青少年体育和健康，人口发展战略与人口高质量发展，建设生育友好型社会，当代中国婚姻家庭问题，就业促进与社会保障，畅通社会流动渠道，新时代青年发展，互联网条件下的青年研究，铸牢中华民族共同体意识，中国特色社会主义宗教理论，边疆治理与边疆民族历史，推进国家安全体系现代化

## 工作流（固定三步，不可跳过）

### Step 1：选议题

- 从上述议题库中**随机或有策略地选择一个** topic。
- 若用户已指定，则直接使用用户指定的 topic。

### Step 2：确定三要素

针对选定的 topic，明确以下三项：

| 要素 | 说明 |
|:-----|:-----|
| **具体问题** | 该议题中一个具体的经验谜题或矛盾现象，包含现实痛点与关键数据 |
| **研究视角** | 从哪个学科/理论传统切入，核心概念是什么 |
| **预期发现** | 用反常识方式揭示深层机制（"不是...而是..."） |

**约束**：
- 具体问题必须是可经验检验的，必须包含现实中的张力或悖论。
- 研究视角必须是社会科学领域内有理论传统的学科视角（如政治社会学、法律社会学、经济社会学、公共管理学、传播学、人口学等）。
- 核心概念 1–3 个，必须来自既有理论文献，避免生造。

### Step 3：写成一段话（固定五句结构）

将 Step 2 的三要素整合为**一段完整的中文文字**，严格遵循以下五句叙事结构，**不可调换顺序，不可遗漏任何一句**：

1. **第一句 — 现实痛点 + 关键数据**：用一个具体的经验现象开篇，点明规模、群体或数据，制造问题意识。句式参考：「...催生的...正陷入...的困境，...导致...」。
2. **第二句 — 制度悖论 / 理论张力**：揭示旧制度框架与新现实之间的结构性冲突。**必须用因果承接词（如"问题的根源在于""症结在于"）与第一句咬合**，形成"现象→根源"推进。句式参考：「问题的根源在于：现有...因固守...，导致...」或「...与...之间的断裂构成了...」。
3. **第三句 — 独特视角 + 核心概念**：明确跳出什么传统视角，引入什么理论概念。**引入概念时必须即时锚定——用破折号"——"或"即"当场解释该概念在本文中的具体含义**，把抽象概念嵌入到经验语境中，避免概念悬空。句式参考：「本研究跳出传统的...视角，引入...核心概念"..."——即...，揭示...的深层机制。」
4. **第四句 — 预期震撼发现**：用反常识的判断揭示核心机制。**必须让第三句引入的核心概念直接参与本句的机制叙述**，形成概念咬合，而非简单重复问题。句式参考：「研究预期发现：...并非...，而是...（核心概念）...导致...」或「这一悖论并非...，而是...（核心概念）...的逻辑使然。」
5. **第五句 — 研究意义**：说明为哪个政策或理论议题提供支撑。句式参考：「本成果将为...提供关键理论支撑。」或「这对...具有重要启示。」

**形式要求**：
- **字数严格控制在 300 字以内**（含标点）。
- **一段连续中文文本，不列点，不分段，不加任何小标题**。
- **措辞必须有硬度**：避免"削弱""影响""不足"等疲软表达，优先使用"制造""消解""挤压""强制""滑向"等硬朗动词；第四句的反常识判断要有冲击力，第五句避免平淡的"提供理论支撑"，改用"打破...悖论""构建...框架"等有政策张力的表达。
- 文字风格为学术提案/选题说明，逻辑递进，层层收紧。

## 输出格式示例

**选定议题**：数智社会治理

**输出**：

> 随着算法嵌入基层治理，数智手段已覆盖超八成城乡社区，但技术赋权却带来"数字悬浮"——系统越智能，离群众诉求越远。问题的根源在于：现有数智治理固守"技术中立"逻辑，将复杂社会矛盾压缩为可计算指标，与基层"情境化回应"需求形成断裂。本研究跳出"技术效率"视角，引入"技术官僚化"概念——即行政权力借技术将社会问题标准化处理、同时挤压基层自由裁量空间的过程，揭示算法治理如何通过消解基层回应性而制造系统性失效。研究预期发现：数智治理失灵并非技术不成熟，而是"技术官僚化"将群众诉求强制降维为可排序指标，使基层从"解决问题"滑向"完成指标"。本成果将为打破"数智"与"善治"的悖论、构建回应导向的算法治理框架提供关键理论依据。

（字数：298 字）

## 稳定性规则（必须遵守）

1. **每次输出只生成一段研究设想**，不生成多个备选。
2. **不展开文献综述**，不列举具体学者姓名与年份。
3. **不添加 Step 1/2/3 的标记或说明文字到最终输出中**，只输出最终那一段。
4. **若用户未指定议题，则每次调用时从议题库中随机或按轮换顺序选取一个**，确保不同调用之间 topic 不重复。
5. **字数超限必须自动压缩**，优先压缩第五句（意义）和第一句（铺陈），保留第二至第四句的核心逻辑。
6. **禁止输出表格、列表、流程图**，最终输出必须为一段连续的中文文本。
7. **五句结构不可改动**：第一句痛点数据 → 第二句制度悖论 → 第三句视角概念 → 第四句预期发现 → 第五句研究意义。即使字数紧张，也必须保留五句的骨架，可通过压缩每句长度来适配。

## 常见错误与规避

| 错误类型 | 表现 | 规避方法 |
|:---------|:-----|:---------|
| 五句结构残缺 | 漏掉第二句或第四句，导致逻辑断裂 | 输出前检查是否五句齐全，顺序正确 |
| 视角模糊 | 「从多学科视角进行研究」——等于没有视角 | 只选一个主导学科视角，明确核心理论传统 |
| 概念堆砌 | 罗列五六个概念，缺乏主线 | 只保留 1–3 个最紧密关联的核心概念 |
| 第四句平庸 | 只是重复问题描述，没有反常识发现 | 必须用"不是...而是..."揭示深层机制 |
| 概念悬空 | 第三句抛出理论概念后，第四句不再提及 | 让核心概念在第四句直接参与机制叙述，形成咬合 |
| 视角割裂 | 从经验现象跳到理论视角缺乏过渡 | 第二句用因果承接词（"问题的根源在于"）建立衔接 |
| 措辞疲软 | 第四句用"削弱""影响"，第五句用"提供理论支撑" | 改用"制造""消解""强制"等硬朗动词；第五句用"打破...悖论""构建...框架" |
| 字数超标 | 第一句背景铺陈过多 | 砍掉修饰语，保留"主体+困境+数据" |
