# Idea-to-Perspective：从选题说明到读书报告的视角锚定

## Purpose

将学生的研究设想转化为一份经过精心策划的阅读任务，迫使其真正接触学术传统。在已有选题或研究设想的基础上，识别竞争性的理论透镜，将每个透镜映射到经典学者及其代表作，并指导学生撰写读书报告，将作者的理论框架嵌入自身的研究问题。

支持**双模式输出**：
- **教学版（teaching）**：面向本科生，提供 3 组竞争性学者-书籍候选，学生选择后生成定制化读书报告任务单。核心目标是防模板化、迫使学生真正阅读。
- **申报版（application）**：面向课题申报者，基于申请人前期研究积累，诊断最适合的理论传统，推荐 1 个深度匹配的权威学者/著作，生成理论框架深化方案。核心目标是确保理论视角与前期积累有延续性、提升评审专家对申请人理论功底的信任。

## Mode Selection

用户未指定模式时，默认输出**教学版**。如果用户明确说"申报""课题""基金""立项""前期成果"，自动切换为**申报版**。

---

## Input Requirements

用户必须提供：

1. **研究设想**（来自 `topic-to-idea` 的输出，或学生自写的 200–400 字研究设想）
2. **学科/课程背景**（可选但建议提供，例如"社会学导论""公共管理"）

**申报版额外需要**：
3. **前期研究方向**（申请人过去 3-5 年的核心研究领域，如"社区治理""数字政府""组织社会学"）
4. **已读过的经典著作**（申请人已深入阅读过的 2-3 本理论著作，如"读过福柯《规训与惩罚》、韦伯《经济与社会》"）

### 【教学版】四步流程

#### Step 1: Deconstruct the Proposal

分析学生的研究设想，提取以下要素：

| 项目 | 说明 |
|:-----|:-----|
| Core phenomenon | 研究的具体社会现象 |
| Implicit causal claim | 设想中隐含的因果关系 |
| Already-used concepts | 已使用的理论概念（1–3 个） |
| Theoretical gap | 这些概念是否被锚定还是悬空 |
| Three competing dimensions | 三个能为同一现象提供*竞争性*解释的理论传统 |

关键规则：三个维度必须代表真正的理论竞争（不同的因果判断），而非互补视角。

### 【申报版】三步流程

#### Step A: Theoretical Tradition Diagnosis

分析申请人的研究设想和前期积累，诊断其理论基因：

| 诊断项 | 说明 |
|:-------|:-----|
| 前期理论传统 | 申请人已深入阅读过的理论著作属于哪个传统？（如福柯→后结构主义/治理性；韦伯→理性化/官僚制；波兰尼→经济社会学/大转型） |
| 经验领域 | 申请人长期关注哪个经验领域？（如社区治理、数字政府、劳动过程） |
| 概念工具 | 申请人前期研究中已熟练使用的核心概念是什么？ |
| 理论缺口 | 当前设想中的概念是否已锚定到具体著作？是否需要深化或替换？ |

**诊断输出模板**：
```
【理论基因诊断】
前期理论传统: [传统名称]
经验领域: [领域]
已熟练概念: [概念1], [概念2]
当前设想的理论缺口: [缺口描述]

【匹配建议】
基于以上诊断，建议从 [传统A] 深化，而非另起炉灶选择 [传统B/C]。
理由：申请人已有 [具体阅读/研究经历] 的积累，选择 [传统A] 可以形成
"经验深耕 + 理论延续"的申报优势，让评审专家感受到申请人的学术连续性。
```

#### Step B: Recommend One Deeply Aligned Scholar-Book Pair

基于诊断结果，推荐 **1 个**（而非 3 个）深度匹配的学者-书籍配对：

**匹配标准**：
- 学者必须是该传统内的**经典人物**（非 trendy 学者）
- 著作必须是申请人的**前期阅读可自然延伸**的作品（如已读福柯《规训与惩罚》，推荐《安全、领土与人口》而非转向斯科特）
- 著作的核心理论必须能**直接深化**申请人的研究问题，而非仅仅"相关"
- 优先选择**有中文译本**且**页数适中**（<400 页）的著作

**输出格式**：
```
【推荐视角: 理论传统名称 + 学者姓名】

匹配度评估: ⭐⭐⭐⭐⭐（5星为最佳匹配）

Scholar: [姓名] ([生卒年/国籍/学派])
Book: 《[书名]》([初版年份]) [译者/出版社/中译年份]

为什么推荐这一本（而非其他）:
1. [与前期阅读的延续性]
2. [与研究问题的深化关系]
3. [评审专家视角：显示申请人的理论功底]

核心论点（≤100 字）:
[一句话概括]

如何深化你的研究（≤100 字）:
[具体说明此书如何帮助申请人从"描述现象"上升到"理论解释"]

申报策略提示:
- 在活页"选题依据"中，将此书作为理论框架的核心锚点
- 在"前期成果"中，如有相关阅读笔记或课程论文，可作为前期积累展示
- 避免在活页中写"本书启发了我"，而应写"本书的[概念]为理解[现象]提供了[机制]"
```

#### Step C: Generate Theory-Framework Deepening Plan

为申报场景生成理论框架深化方案：

1. **概念操作化建议**：如何将书中的核心概念转化为可测量的研究变量
2. **文献对话策略**：如何在活页中展示申请人与该理论传统的深度对话
3. **竞争理论排除**：如何简要说明为什么不选择其他两个竞争性传统（显示申请人的理论判断力）

**输出模板**：
```
【理论框架深化方案】

1. 概念操作化
   核心概念: [概念名]
   操作化方向: [如何从抽象概念转化为可观察的经验指标]
   示例: [给出一个具体的操作化示例]

2. 文献对话策略
   在活页中的位置: [选题依据的哪个部分]
   叙述策略: [如何展示对话深度，而非简单引用]
   模板句: "[学者] 的 [概念] 揭示了 [机制]，但在 [中国情境/特定现象] 中，
           这一机制表现为 [具体形态]，现有研究尚未充分展开这一分析。"

3. 竞争理论排除（展示理论判断力）
   需要排除的理论: [理论B、理论C]
   排除理由（每理论一句话）:
   - [理论B]: [为什么不适合本研究]
   - [理论C]: [为什么不适合本研究]
   注意: 排除时要尊重对手，不可贬低，只需说明"不适合"而非"错误"
```

## 教学版稳定性规则

1. 必须先完成 Step 1 再进入 Step 2。不要在未诊断设想理论缺口之前推荐书籍。
2. 三对配对必须代表真正的竞争关系。
3. 每对配对必须与学生设想的至少一个具体元素建立连接。
4. 至少有一对须挑战学生设想的默认假设。
5. 不要推荐教材、百科全书或综述文章。
6. 如果学生的设想过于模糊，先提出一个澄清问题。
7. 在学生明确选择一对之前，不要生成完整的读书报告任务单。

## 申报版稳定性规则

1. **不得推荐 3 个竞争性视角**：申报场景下，推荐 1 个最匹配的视角，展示申请人的理论专注力。
2. **必须与前期积累有延续性**：如果推荐的书籍与申请人前期阅读完全无关，必须说明"虽然此书与前期阅读不同，但它是该传统内最经典的入门著作，建议优先阅读"。
3. **必须考虑著作的可及性**：优先推荐有中文译本、图书馆可借到的著作。
4. **必须生成理论框架深化方案**：不仅仅是推荐一本书，而是告诉申请人如何在活页中使用这本书。
5. **不得生成读书报告任务单**：申报版不需要读书报告，需要的是理论框架的论证方案。
6. **如果申请人前期积累为空**，停止并提示："申报版需要您提供前期研究方向和已读经典著作。如果暂无积累，建议先使用教学版完成 1-2 轮经典阅读后再进入申报。"

## 常见错误与修复

| Error | Manifestation | Fix |
|:------|:--------------|:----|
| 教学版：Fake rivalry | 三本书实际上意见一致 | 检查：它们是否对同一现象做出了不同的因果主张？ |
| 教学版：Generic matching | 书籍虽有名，但与选题仅有模糊关联 | 确保"Connection to proposal"明确具体概念对应关系 |
| 申报版：推荐与前期无关的学者 | 申请人读过福柯，推荐斯科特 | 先诊断理论基因，推荐同一传统内的深化读物 |
| 申报版：推荐 trendy 学者 | 推荐近 5 年的热门学者而非经典 | 坚持经典优先，经典著作更能显示理论功底 |
| 申报版：缺乏操作化建议 | 只推荐书，不告诉如何在活页中使用 | 必须生成理论框架深化方案 |
| 申报版：贬低竞争理论 | "韦伯已经过时了，福柯才是对的" | 排除竞争理论时要尊重对手，说明"不适合"而非"错误" |
