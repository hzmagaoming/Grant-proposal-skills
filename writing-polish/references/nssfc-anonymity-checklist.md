# 国家社科基金活页匿名合规性 20 项检查清单

> 本清单供 `writing-polish` skill 调用。目标：确保活页通过 NSSFC 系统的匿名评审，不暴露申请人任何身份信息。

---

## 高风险项（必须修改，否则直接淘汰）

### A1. 第一人称暴露
- [ ] 全文无"笔者""我的研究""我发现""我提出"等第一人称表述
- [ ] 全文无"我们团队""本课题组"等复数第一人称
- [ ] 全文无"本人""本申请人"等指向性表述

**替换规则**：
- "笔者发现" → "研究发现"
- "我的选题" → "本研究"
- "我在某城市调研时发现" → "已有研究表明"

### A2. 学校/单位名称暴露
- [ ] 无学校全称或简称（如"浙江大学""北大""清华"）
- [ ] 无院系名称（如"公共管理学院""社会学系"）
- [ ] 无单位特有项目名称（如"某某大学某某研究中心"）
- [ ] 无单位所在地的标志性建筑或地标（如"未名湖畔""珞珈山下"）

**替换规则**：
- "浙江大学公共管理学院" → "某高校"
- "我在北京调研时发现" → "在某城市调研时发现"

### A3. 导师/合作者姓名暴露
- [ ] 无导师姓名（包括直接提及和间接引用"导师的建议"）
- [ ] 无合作者姓名（即使作为参考文献作者，也要注意前期成果中的署名）
- [ ] 无导师的标志性理论术语（如果该术语几乎仅为导师专用，可能暴露身份）

### A4. 前期成果署名暴露
- [ ] 前期成果列表中，作者姓名已替换为"课题负责人"或删除
- [ ] 前期成果的年份-期刊组合不会罕见到可识别（如某年某期刊仅发表了 1 篇该主题论文）

---

## 中风险项（建议修改，可能引发评审专家猜测）

### B1. 地域暴露
- [ ] 无申请人所在城市特有的地名（如"钱江新城""中关村""陆家嘴"）
- [ ] 无只有某地才有的政策名称（如"某市特有的网格化管理平台名称"）
- [ ] 无过于具体的行政区划（如"某某区某某街道"，建议改为"某城市某社区"）

### B2. 数据暴露
- [ ] 无只有本单位才有的数据库名称（如"某某大学中国调查与数据中心"）
- [ ] 无只有本单位才有的调查项目名称（如"某某大学 2023 年基层治理调查"）
- [ ] 无内部档案编号或内部系统名称

### B3. 技术暴露
- [ ] 无过于个性化的技术路线描述（如"基于我开发的 Python 脚本"）
- [ ] 无个人 GitHub/知乎/公众号等可追踪的账号信息
- [ ] 无只有申请人使用过的特定研究方法组合

### B4. 时间暴露
- [ ] 无过于个人化的时间线索（如"2024 年暑期我在某单位实习时发现"）
- [ ] 无只有申请人经历过的特定事件（如"在某次国际会议上与某学者的讨论启发了我"）

### B5. 成果暴露
- [ ] 前期成果数量适中（3-5 项），不会太少（显得没基础）也不会太多（显得凑数）
- [ ] 前期成果的年份分布合理，不会集中在某一年（暗示赶工）
- [ ] 前期成果的期刊层次与申请人身份匹配，不会明显过高或过低

---

## 低风险项（可选修改，锦上添花）

### C1. 致谢暴露
- [ ] 无致谢语（活页中不应出现致谢）
- [ ] 无"感谢某某基金资助"等暗示前期身份的信息

### C2. 脚注暴露
- [ ] 脚注中无个人联系方式、单位地址
- [ ] 脚注中无"参见本人的另一项研究"等自我引用

### C3. 语言风格暴露
- [ ] 无过于个性化的写作风格（如特有的口头禅、标志性的句式结构）
- [ ] 无只有某学校某导师门下的学生才会使用的术语翻译或概念界定

### C4. 选题偏好暴露
- [ ] 选题方向不会过于狭窄到与申请人的博士论文/博士后报告完全重合
- [ ] 选题不会使用只有某小圈子才关注的经验材料

---

## 检查流程

**Step 1: 关键词全局搜索**

在活页中搜索以下关键词，逐一排查：
```
我、笔者、本人、我们、本课题组、导师、指导教师、学校、大学、学院、系、研究所、中心
```

**Step 2: 地名扫描**

检查所有地名，判断是否过于具体：
- 省级：安全
- 市级：通常安全，但如果申请人所在城市是特大城市且研究对象也是该城市，需谨慎
- 区/县级：中风险，建议模糊化
- 街道/社区级：高风险，必须模糊化

**Step 3: 前期成果审查**

逐条检查前期成果：
- 是否与选题直接相关？（无关的应删除）
- 是否可能通过"年份+期刊+主题"组合反推身份？
- 是否需要增加 1-2 篇其他期刊论文稀释识别度？

**Step 4: 逻辑一致性检查**

- 如果活页中声称"基于某内部数据库"，但前期成果中没有相关数据使用经验，评审专家可能怀疑数据来源的真实性
- 如果活页中声称"长期田野调查"，但前期成果中没有相关田野经验，可能暴露"虚报"

---

## 合规评分标准

| 分数 | 状态 | 说明 |
|:-----|:-----|:-----|
| 20/20 | 完美匿名 | 无任何风险项，评审专家完全无法推断身份 |
| 18-19/20 | 安全 | 有 1-2 个低风险项，不影响匿名评审 |
| 15-17/20 | 临界 | 有 1-2 个中风险项，建议修改 |
| 12-14/20 | 危险 | 有 1-2 个高风险项，必须修改 |
| <12/20 | 严重违规 | 多个高风险项，可能直接导致淘汰 |
