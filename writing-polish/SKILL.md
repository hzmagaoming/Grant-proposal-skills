# Writing-Polish：活页润色与三角色模拟评审

## Purpose

本 skill 是教学/申报 skill 流水线的**最终质检环节**。在 `proposal-writing` 输出完整活页后，通过系统化润色和多视角模拟评审，将活页从"可提交"提升至"有竞争力"。

核心功能：
1. **字数精修**：自动检测各章节字数，提供压缩/扩充建议
2. **匿名合规性检查**：20 项深度扫描，识别可能暴露申请人身份的信息
3. **逻辑与一致性检查**：跨章节概念对齐、假设编号一致性、时间/数字一致性
4. **三角色模拟评审**：内行专家 + 邻近学科专家 + 外行专家，还原真实通讯评审场景
5. **综合改进建议**：基于三角色反馈生成优先级排序的修改清单

## Input Requirements

用户必须提供：

1. **完整活页文本**（Markdown 格式，包含 6 个章节 1-6）
2. **目标用途**（课程作业 / 毕业论文开题 / 国家社科基金申报 / 省级课题申报）
3. **学科分类**（如：社会学、公共管理学、政治学等）
4. **理论方向**（如：基层治理、数字治理、国家-社会关系等）

可选提供：
- 申请人前期研究方向（仅用于模拟评审的"基础"维度评估，不写入输出）
- 字数超限的具体章节（如"选题依据写了 3000 字，需要压缩到 2200"）

## Workflow

### Step 0: 解析输入与确认目标

从用户输入中提取：

| 项目 | 提取内容 | 确认方式 |
|------|----------|----------|
| 活页文本 | 完整 6 章内容 | 直接读取 |
| 目标用途 | 课程 / 开题 / 国社科 / 省社科 | 用户声明或推断 |
| 学科分类 | 一级学科 | 用户声明或从选题推断 |
| 理论方向 | 核心研究领域 | 从选题说明推断 |

**如果输入缺少活页文本**，停止并提示：
> "本 skill 需要您提供完整的活页文本（6 个章节）。请提供后重新调用。"

**目标用途确认模板**：
```
【已确认的目标】
目标用途：[课程作业 / 开题报告 / 国家社科基金申报]
学科分类：[学科]
理论方向：[方向]
字数上限：[7000 字 / 不限]

如果以上信息有误，请告知修正后我们再继续。
```

### Step 1: 字数精修

**教学版（课程作业）**：
- 检查总字数是否超过教师要求（通常 5000-8000 字）
- 各章节比例是否合理
- 提供压缩/扩充建议

**申报版（国家社科基金）**：

| 章节 | 建议字数 | 弹性范围 | 当前字数 | 状态 |
|:-----|:---------|:---------|:---------|:-----|
| 1. 选题说明 | 250-300 | ≤300（硬性） | [统计] | [✅/⚠️/❌] |
| 2. 选题依据 | 2000-2500 | 可压缩至 1800 | [统计] | [✅/⚠️/❌] |
| 3. 研究内容 | 1500-2000 | 可压缩至 1200 | [统计] | [✅/⚠️/❌] |
| 4. 创新之处 | 600-800 | 可压缩至 500 | [统计] | [✅/⚠️/❌] |
| 5. 预期成果 | 200-400 | 可压缩至 200 | [统计] | [✅/⚠️/❌] |
| 6. 参考文献 | 400-600 | 不计入主体字数 | [统计] | [✅/⚠️/❌] |
| **总计** | **≤7000** | **硬性上限** | **[统计]** | **[✅/⚠️/❌]** |

**如果字数超限**，按以下优先级提供压缩方案：
1. **优先压缩"选题依据"**：删减文献细节论证，保留核心论点（每篇文献保留"主张→挑战→盲区"3句话）
2. **其次压缩"研究内容"**：保留框架和逻辑，删减具体操作步骤（如"第一周下载文件"等）
3. **绝不压缩"创新之处"**：这是评审的核心关注点，宁可扩充其他章节也不要压缩此处
4. **最后压缩"预期成果"**：保持简洁，每项 1-2 句话

**压缩技巧模板**：
```
【压缩建议：选题依据】
- 原文："Tang (2020) 基于2011至2016年多个城市的田野资料发现，网格化治理通过吸纳中产阶层居民志愿者、激活退休精英的政治影响力，以及建立社区党组织，有效强化了国家对城市邻里空间的渗透与规制。"
- 压缩："Tang (2020) 主张网格化增强了国家基层渗透能力，但忽视了指标生产政治。"
- 节省字数：约 60 字
```

### Step 2: 匿名合规性深度检查

加载 `references/nssfc-anonymity-checklist.md` 执行 20 项扫描。

**检查维度**：

| 维度 | 检查项 | 风险等级 |
|:-----|:-------|:---------|
| **人称暴露** | "笔者""我的研究""我们团队" | 🔴 高 |
| **单位暴露** | 学校名称、院系名称、单位特有项目名称 | 🔴 高 |
| **地域暴露** | 申请人所在城市特有的地名、区域政策 | 🟡 中 |
| **数据暴露** | 只有本单位才有的数据库名称、调查项目名称 | 🟡 中 |
| **导师暴露** | 导师姓名、导师的标志性理论术语 | 🟡 中 |
| **成果暴露** | 前期成果的年份+期刊组合可能暴露身份 | 🟡 中 |
| **时间暴露** | "我在某年某月实习时发现" | 🟢 低 |
| **技术暴露** | 过于个性化的技术路线或代码仓库链接 | 🟢 低 |

**输出格式**：
```
【匿名合规性检查报告】

高风险项（必须修改）：
- [ ] 第 X 章出现"笔者"，应改为"本研究"
- [ ] 第 X 章提到"浙江大学"，应改为"某高校"或删除

中风险项（建议修改）：
- [ ] 第 X 章前期成果"2023年《社会学研究》"组合较罕见，建议增加1-2篇其他期刊论文稀释

低风险项（可选修改）：
- [ ] 第 X 章提到"2024年暑期调研"，过于个人化，建议改为"已有研究表明"

匿名合规评分：[X/20]（≥18 为安全，15-17 为临界，<15 为危险）
```

### Step 3: 逻辑与一致性检查

**跨章节概念对齐**：
- [ ] 选题说明中的"核心概念"与研究内容中的"概念界定"是否一致？
- [ ] 选题依据中的"理论缺口"与研究内容中的"研究问题"是否对应？
- [ ] 文献综述中的"假设"与研究内容中的"目标"是否一致？

**数字一致性**：
- [ ] 时间描述是否一致（如"2020-2024"在全文各处是否相同）？
- [ ] 样本量描述是否一致（如"6 个城市"在各章是否相同）？
- [ ] 经费/金额描述是否一致？

**假设编号一致性**：
- [ ] 文献综述中的假设编号（H1/H2/H3）与研究内容中提到的假设是否一致？
- [ ] 创新之处中提到的假设是否与文献综述一致？

**输出格式**：
```
【逻辑一致性检查报告】

概念对齐：
- ✅ 选题说明"清晰化" = 研究内容"清晰化" = 创新之处"清晰化"
- ⚠️ 选题说明用"治理性"，研究内容用"清晰化"，建议统一为"清晰化"

数字一致性：
- ✅ "6 个城市"在各章一致
- ❌ 第 2 章写"2020-2024"，第 3 章写"2019-2024"，请统一

假设一致性：
- ✅ H2 在文献综述、研究内容、创新之处中编号一致
```

### Step 4: 三角色模拟评审

基于用户提供的学科分类和理论方向，构建三个评审角色。每个角色独立评审，互不干扰。

**角色构建原则**：
- **内行专家**：熟悉用户提供的理论方向，关注理论原创性和方法严谨性
- **邻近学科专家**：熟悉用户提供的学科分类，但不专门研究该理论方向，关注跨学科价值和可读性
- **外行专家**：社科领域学者，不熟悉本方向和学科，关注问题意识清晰度和社会意义

**评审维度与权重**（严格遵循国家社科基金通讯评审意见表）：
- 选题（30%）：学术价值、现实意义、问题意识
- 论证（50%）：文献梳理、研究内容、创新之处、可行性
- 基础（20%）：前期积累、团队条件、数据/资料可及性

---

#### 角色 A：内行专家（Subfield Expert）

**身份设定**：
> 您是本领域深耕 10 年以上的学者，熟悉 [理论方向] 的中英文核心文献，曾主持多项国家级课题，对 [学科] 的理论前沿和方法论争议有深入把握。您的评审风格：直击要害，不客气，关注理论原创性和因果识别的严谨性。

**评审输出**：
```
【角色 A：内行专家】

总体印象：[一句话概括对活页的整体感受，如"选题新颖，但方法设计略显保守"]

评分（满分 10 分）：
- 选题（30%）：[X]/10 → 加权 [X×0.3]
- 论证（50%）：[X]/10 → 加权 [X×0.5]
- 基础（20%）：[X]/10 → 加权 [X×0.2]
- 加权总分：[X]/10

三个亮点：
1. [具体指出理论、方法或文献对话上的亮点]
2. ...
3. ...

三个疑问/担忧：
1. [具体指出理论漏洞、方法缺陷或文献盲区]
2. ...
3. ...

关键建议：[最重要的 1 条修改建议，通常是方法或理论层面的]
```

**内行专家常见问题库**（根据理论方向动态选择）：
- "这个概念的测量方案是否可靠？"
- "这个因果机制是否排除了竞争性解释？"
- "文献综述是否遗漏了 [某学派/某学者] 的关键工作？"
- "理论框架与经验材料之间的匹配度如何？"
- "样本选择是否存在系统性偏差？"

---

#### 角色 B：邻近学科专家（Adjacent Discipline Expert）

**身份设定**：
> 您是具有 [学科分类] 背景但不专门研究 [理论方向] 的学者，您的研究领域与选题有交叉（如数字治理、公共政策、组织社会学等），但不熟悉本方向的专有文献。您关注：理论框架是否可迁移、跨学科对话潜力、论证逻辑是否自洽、概念是否对非专业人士清晰。

**评审输出**：
```
【角色 B：邻近学科专家】

总体印象：[一句话概括，如"问题意识清晰，但理论术语需要更多解释"]

评分（满分 10 分）：
- 选题（30%）：[X]/10 → 加权 [X×0.3]
- 论证（50%）：[X]/10 → 加权 [X×0.5]
- 基础（20%）：[X]/10 → 加权 [X×0.2]
- 加权总分：[X]/10

三个亮点：
1. [从跨学科角度指出的亮点]
2. ...
3. ...

三个疑问/担忧：
1. [概念不清、逻辑跳跃、术语未解释等问题]
2. ...
3. ...

关键建议：[最重要的 1 条修改建议，通常是概念澄清或逻辑补全]
```

**邻近学科专家常见问题库**：
- "'米提斯'概念与'社会资本''地方性知识'的区别是什么？"
- "这个理论视角在其他国家的治理场景中是否同样适用？"
- "研究方法是否足以支撑理论野心？"
- "预期成果的形式是否与学科惯例匹配？"

---

#### 角色 C：外行专家（Generalist / Outsider）

**身份设定**：
> 您是社科领域的学者（如经济学、历史学、教育学背景），但不熟悉 [学科分类] 和 [理论方向]。您评审国家社科基金时，首先看"这个问题为什么值得资助""研究结论对非学术读者有什么价值""语言是否通俗易懂"。您对过于技术化的论证缺乏耐心。

**评审输出**：
```
【角色 C：外行专家】

总体印象：[一句话概括，如"问题很重要，但读起来有点费劲"]

评分（满分 10 分）：
- 选题（30%）：[X]/10 → 加权 [X×0.3]
- 论证（50%）：[X]/10 → 加权 [X×0.5]
- 基础（20%）：[X]/10 → 加权 [X×0.2]
- 加权总分：[X]/10

三个亮点：
1. [从社会意义、现实关怀、可读性角度指出的亮点]
2. ...
3. ...

三个疑问/担忧：
1. [问题意识不清、社会意义不显著、语言晦涩等问题]
2. ...
3. ...

关键建议：[最重要的 1 条修改建议，通常是问题意识强化或语言简化]
```

**外行专家常见问题库**：
- "这个问题为什么重要？普通人能理解吗？"
- "研究结论对政策制定者有什么实际帮助？"
- "为什么要用这么复杂的理论来解释一个看起来简单的问题？"
- "3 年后结项时，我能看到什么具体成果？"

## Stability Rules

1. **三角色必须独立评审**：角色 A 的评审不应受角色 B/C 影响，反之亦然。每个角色基于自身的知识背景和关注焦点给出独立判断。
2. **评分必须严格遵循 3:5:2 权重**：不能随意调整权重。
3. **匿名检查必须覆盖 20 项**：不能遗漏高风险项。
4. **字数统计必须精确**：中文字符、标点、空格均计入（与 NSSFC 系统统计方式一致）。
5. **如果用户未提供学科/理论方向**，先询问再执行模拟评审，不能随意假设。
6. **修订版活页仅在用户明确要求时输出**：默认只输出评审报告。

## Common Errors to Avoid

| Error | Manifestation | Fix |
|:------|:--------------|:----|
| 角色同质化 | 三个角色的评价过于相似，失去模拟价值 | 严格按角色设定独立评审，内行关注方法，外行关注意义 |
| 匿名检查流于形式 | 只检查"笔者""我的"，遗漏地域/数据/导师暴露 | 使用 20 项清单逐项扫描 |
| 字数统计不准确 | 只统计汉字，忽略标点和空格 | 与 NSSFC 系统保持一致，全部计入 |
| 修改建议过于笼统 | "加强理论深度""完善方法设计" | 给出具体到句子的修改示例 |
| 忽视目标用途差异 | 对课程作业和国社科申报使用同一标准 | 先确认目标，再调整评审严格度 |
