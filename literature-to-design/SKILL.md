# Literature-to-Design：从文献综述到研究设计方案

## Purpose

本 skill 是 `perspective-to-literature` 的下游配套工具。学生已经完成了批判性文献综述并选定了假设；现在需要将其转化为一个**可直接写入课题申报活页或开题报告的研究设计方案**。

与面向职业研究者的完整研究设计方案不同，本 skill 专门为**教学场景**和**申报场景**提供差异化设计：

支持**双模式输出**：
- **教学版（teaching）**：本科生可执行、技术门槛低、纯网络数据采集、低成本、6-8 周完成。核心目标是让学生体验完整的研究设计过程。
- **申报版（application）**：面向国家社科基金/课题申报，方法严谨、3 年研究周期、允许混合方法、强调因果识别、数据权威性、团队条件和可行性论证。核心目标是让评审专家相信"这个研究做得成、值得做"。

## Mode Selection

用户未指定模式时，默认输出**教学版**。如果用户明确说"申报""课题""基金""立项""活页"，自动切换为**申报版**。

---

## 输出格式（严格遵循）

本 skill 的输出格式严格匹配课题申报活页标准：

```
3.1 [研究内容]
    课题研究的主要目标、重点难点、整体框架、研究计划及其可行性等。
    （框架思路要列出提纲或目录）

3.2 [创新之处]
    在学术观点、研究方法等方面的特色和创新。
```

**不输出**：数据分析结论、假设检验结果、回归表格、图表。

---

## Arguments

用户已提供：`$ARGUMENTS`

解析：
- 文献综述文件路径（Markdown），或读书报告/选题说明的内容
- 目标用途（课程作业、毕业论文开题、课题申报）
- 可选约束（时间限制、数据限制、技术限制）
- **申报版额外需要**：团队构成、前期数据积累、合作单位支持等信息

## Workflow（双版本）

### Step 0: 读取输入（双版本通用）

读取用户提供的文献综述或读书报告文件，提取以下关键信息：

| 提取项 | 说明 | 如果缺失 |
|--------|------|----------|
| 研究问题（RQ） | 核心经验问题 | 从假设反推 |
| 假设（H1/H2/H3） | 用户已选定的假设 | 停止，提示用户先完成 perspective-to-literature |
| 理论视角 | 学生已选的学者/著作/理论 | 从读书报告中提取 |
| 核心机制 | 假设背后的因果链 | 从文献综述的"解释缺口"中提取 |
| 关键概念 | 2-3个核心分析概念 | 从理论框架中提取 |
| **申报版额外** | 团队构成、前期数据、合作支持 | 从用户输入中提取 |

**输入确认模板**：
```
【已确认的研究要素】
研究问题：[RQ]
选定假设：[H1陈述]
理论视角：[学者/著作/理论]
核心机制：[一句话因果链]
关键概念：[Concept1], [Concept2]

如果以上信息有误，请告知修正后我们再继续。
```

### 【申报版】Step 1: 设计数据采集方案（混合方法、权威数据）

**核心原则**：申报版的数据采集方案必须满足"四性"标准

| 标准 | 要求 | 反面示例 |
|:-----|:-----|:---------|
| **权威性** | 数据来源须为官方统计、权威调查、核心期刊、政府档案 | 不依赖社交媒体帖子、网络论坛等非权威来源 |
| **可及性** | 数据获取路径清晰，最好已获得访问权限或合作意向 | 不依赖"拟申请""拟联系"等不确定来源 |
| **匹配性** | 数据变量必须直接对应研究假设中的核心概念 | 不采集与研究问题无关的"鸡肋变量" |
| **伦理性** | 涉及人类被试的研究须通过伦理审查 | 不规避 IRB/伦理委员会审查 |

**推荐的数据采集策略**（按优先级排序）：

| 优先级 | 数据来源 | 适用场景 | 权威性 |
|:-------|:---------|:---------|:-------|
| P0 | 政府统计年鉴 / 民政部门公开数据 | 宏观趋势分析 | ⭐⭐⭐⭐⭐ |
| P0 | 学术调查数据库（CGSS、CFPS、CHARLS 等） | 个体层面分析 | ⭐⭐⭐⭐⭐ |
| P1 | 政策文本 / 政府档案 / 会议纪要 | 制度分析 | ⭐⭐⭐⭐ |
| P1 | 半结构化访谈 / 焦点小组 | 机制探索 | ⭐⭐⭐⭐ |
| P2 | 问卷调查（分层抽样） | 大样本检验 | ⭐⭐⭐ |
| P2 | 媒体报道 / 社交媒体（辅助性） | 舆情分析 | ⭐⭐ |

**申报版数据采集方案必须包含**：
1. **数据来源**（具体的统计年鉴、数据库、档案馆的名称和年份）
2. **采集字段**（明确列出要采集哪些变量，及变量的操作化定义）
3. **采集方法**（官方下载 / 申请使用 / 自行采集）
4. **样本策略**（分层抽样 /  purposive 抽样 / 滚雪球抽样，及样本量计算的统计依据）
5. **预期样本量**（需要简单的统计依据，如功效分析或同类研究的样本量参考）
6. **时间窗口**（何时采集，为什么选这个时间）
7. **伦理审查**（如果涉及人类被试，说明已通过或将通过哪个伦理委员会的审查）
8. **数据管理**（如何存储、备份、共享，是否符合科研数据管理规范）

**自然实验锚点的设计**（申报版必须有）：

申报版强调因果识别，必须嵌入至少一个**自然实验或准实验设计**：

| 锚点类型 | 示例 | 适用性 |
|:---------|:-----|:-------|
| 政策冲击（DID） | 某城市 2020 年推行网格化改革，邻近城市未推行 | ⭐⭐⭐⭐⭐ |
| 地理断点（RD） | 网格化管理在行政区划边界两侧的差异 | ⭐⭐⭐⭐ |
| 时间断点（ITS） | 网格化改革前后同一地区的调解质量变化 | ⭐⭐⭐⭐ |
| 工具变量（IV） | 某外生变量（如中央财政转移支付）影响网格化投入但不直接影响调解质量 | ⭐⭐⭐ |

**方案设计检查**：
- [ ] 数据采集是否基于权威来源？
- [ ] 是否有自然实验或准实验锚点？（申报版必备）
- [ ] 样本量是否有统计依据？
- [ ] 是否已通过或计划通过伦理审查？
- [ ] 核心变量的操作化定义是否清晰？

### 【申报版】Step 2: 设计分析方法（混合方法、因果识别）

**核心原则**：分析方法必须与理论假设的因果层次匹配

| 假设类型 | 推荐方法 | 不适合的方法 |
|:---------|:---------|:-------------|
| H1 主效应（相关/差异） | OLS 回归 / 逻辑回归 / 组间差异检验 | 纯描述统计 |
| H2 机制检验（中介/调节） | 中介效应模型 / 调节效应模型 / 结构方程模型 | 简单相关分析 |
| H3 稳健性检验（情境边界） | 子样本分析 / 交互项模型 / 安慰剂检验 | 无 |
| 因果识别（自然实验） | DID / RD / IV / PSM-DID | 简单 OLS（内生性未处理） |

**申报版分析方案必须包含**：
1. **因变量**如何测量（操作化定义 + 数据来源）
2. **核心自变量**如何测量（操作化定义 + 数据来源）
3. **控制变量**有哪些（为什么需要控制，及数据来源）
4. **识别策略**：如何处理内生性？（自然实验 / 工具变量 / 固定效应等）
5. **稳健性检验**：至少 2 种稳健性检验方案
6. **每步分析的回答什么问题**：明确说明每个统计检验对应哪个假设
7. **预期结果的方向**：支持/不支持假设时的判断标准
8. **统计软件**：Stata / R / Python（申报版不推荐使用 SPSS）

### 【申报版】Step 3: 撰写 3.1 [研究内容]

严格按照以下结构撰写，不增不减：

#### 3.1.1 课题研究的主要目标

列出 3-4 个具体目标，以问题形式呈现或陈述形式。申报版的目标应该形成"理论→机制→检验→应用"的递进逻辑。

#### 3.1.2 重点难点

| 重点 | 难点 |
|------|------|
| 列出 3 个研究中的关键问题 | 列出 3 个执行中可能遇到的困难 |
| 每个重点附一句话说明为什么重要 | 每个难点附一句话降低难度的方案 |

**申报版额外要求**：
- 难点中必须包含"因果识别"相关的挑战（如"如何处理网格化改革的自选择偏差？"）
- 降低难度的方案必须具体（如"利用 2020 年某政策作为外生冲击，采用双重差分设计"）

#### 3.1.3 整体框架（列出提纲或目录）

以**树形目录**或**章节提纲**形式呈现。

**申报版框架建议**（3 年研究周期）：
```
第一章 导论
  1.1 研究背景与问题提出
  1.2 核心概念界定
  1.3 研究方法与数据来源
  1.4 研究框架与创新点

第二章 文献回顾与理论框架
  2.1 [英文文献梳理]
  2.2 [中文文献梳理]
  2.3 [理论框架构建]

第三章 [核心经验分析 1：描述性发现]
  3.1 [样本描述]
  3.2 [核心变量的分布特征]

第四章 [核心经验分析 2：因果推断]
  4.1 [主效应检验]
  4.2 [机制检验]
  4.3 [稳健性检验]

第五章 [异质性分析与讨论]
  5.1 [情境差异分析]
  5.2 [与现有文献的对话]

第六章 结论与政策建议
  6.1 主要研究发现
  6.2 理论贡献与实践启示
  6.3 研究局限与未来方向
```

#### 3.1.4 研究计划及其可行性

**申报版与教学版的区别**：

| 项目 | 教学版 | 申报版 |
|:-----|:-------|:-------|
| 时间线 | 6-8 周 | 3 年（36 个月） |
| 可行性论证 | "本科生能做" | "申请人有能力做" |
| 团队构成 | 单人 | 课题组 + 分工 |
| 数据来源 | 纯网络公开 | 权威数据库 + 可能的一手采集 |
| 自然实验 | 建议有（可豁免） | 必须有 |
| 伦理审查 | 仅采集公开数据 | 涉及人类被试须通过 IRB |

**申报版时间线示例**（36 个月）：

| 阶段 | 时间 | 任务 | 阶段性成果 |
|:-----|:-----|:-----|:-----------|
| 第一阶段 | 第 1-6 月 | 文献深化、理论框架细化、数据获取 | 工作论文 1 篇 |
| 第二阶段 | 第 7-18 月 | 数据采集与清洗、描述性分析 | CSSCI 论文 1 篇 |
| 第三阶段 | 第 19-30 月 | 因果推断分析、机制检验、稳健性检验 | CSSCI 论文 2 篇 |
| 第四阶段 | 第 31-36 月 | 政策建议撰写、结项报告、专著统稿 | 结项报告 + 专著初稿 |

**申报版可行性论证必须包含**：
1. **数据可及性**：数据已获取 / 已申请 / 已获合作单位支持
2. **方法可控性**：申请人已掌握所需统计方法（通过前期成果展示）
3. **时间可控性**：3 年时间线合理，每个阶段有明确的里程碑
4. **团队支撑**：课题组成员的分工和各自的专业优势
5. **单位支持**：所在单位在数据、经费、设备等方面的支持

### 【申报版】Step 4: 撰写 3.2 [创新之处]

严格按照以下两个维度撰写：

#### 3.2.1 学术观点创新

- **理论视角的独特性**：申请人用了什么非默认的理论，为什么这个理论更适合本研究
- **核心概念的创新**：从理论中提炼出的新概念，或既有概念在新情境中的重新界定
- **与现有文献的对话**：现有文献忽略了什么，本研究如何填补
- **因果机制的揭示**：本研究发现了什么新的因果机制或边界条件

#### 3.2.2 研究方法创新

- **因果识别策略的独特性**：自然实验/准实验设计如何解决了内生性难题
- **数据组合的创新**：如何整合了不同来源的数据（如官方统计+调查数据+政策文本）
- **方法的可复制性**：其他研究者能否用同样方法研究其他问题

**申报版创新之处的写法禁忌**：
- ❌ "填补了国内空白"
- ❌ "首次研究"
- ❌ "具有重要理论意义和现实意义"（空话）
- ✅ "现有文献将网格化视为国家能力建设成功案例，本研究则通过清晰化理论揭示其内在张力……"
- ✅ "与 Tang (2020) 的动员叙事不同，本研究指出……"

## 常见错误与修复

| 错误 | 表现 | 修复 |
|:-----|:-----|:-----|
| 教学版：方法过复杂 | 建议 PSM、DID、工具变量 | 替换为 t 检验、简单回归、描述统计 |
| 教学版：数据采集成本高 | 需要实地核查、访谈、购买数据 | 替换为纯网络方法 |
| 申报版：方法过简单 | 仅用描述统计，无因果识别 | 增加自然实验/准实验设计 |
| 申报版：数据来源不权威 | 依赖社交媒体、网络论坛 | 替换为官方统计、权威调查、政府档案 |
| 申报版：可行性论证空洞 | "时间充足""资料丰富" | 具体说明数据已获取、方法已掌握、团队已组建 |
| 申报版：无自然实验锚点 | 纯横截面描述，无因果推断意识 | 嵌入政策冲击/地理断点/时间断点 |
| 申报版：创新之处写得太虚 | "填补了空白""首次研究" | 改为"与 [某文献] 不同，本研究……" |
| 双版本：包含分析结论 | 写了"预计发现幽灵商户占比 30%" | 删除所有预期结果和结论，只写设计方案 |
