# Literature-to-Design：从文献综述到研究设计方案

## Purpose

本 skill 是 `perspective-to-literature` 的下游配套工具。学生已经完成了批判性文献综述并选定了假设；现在需要将其转化为一个**可直接写入课题申报活页或开题报告的研究设计方案**。

与 `scholar-design`（面向研究者，功能完整、方法复杂）不同，本 skill 专门为**教学场景**设计，核心约束是：
- **本科生可执行**：技术门槛低、可理解、可操作
- **纯网络数据采集**：不需要实地调研、差旅、访谈
- **低成本**：接近零预算
- **靠近准实验**：利用公开事件或自然变异作为因果识别的锚点

## 输出格式（严格遵循）

本 skill 的输出格式严格匹配课题申报活页标准：

```
3.1 [研究内容]
    课题研究的主要目标、重点难点、整体框架、研究计划及其可行性等。
    （框架思路要列出提纲或目录）

3.2 [创新之处]
    在学术观点、研究方法等方面的特色和创新。
```

**不输出**：数据分析结论、假设检验结果、回归表格、图表。

---

## Arguments

用户已提供：`$ARGUMENTS`

解析：
- 文献综述文件路径（Markdown），或读书报告/选题说明的内容
- 目标用途（课程作业、毕业论文开题、课题申报）
- 可选约束（时间限制、数据限制、技术限制）

## Workflow

### Step 0: 读取输入

读取用户提供的文献综述或读书报告文件，提取以下关键信息：

| 提取项 | 说明 | 如果缺失 |
|--------|------|----------|
| 研究问题（RQ） | 核心经验问题 | 从假设反推 |
| 假设（H1/H2/H3） | 用户已选定的假设 | 停止，提示用户先完成 perspective-to-literature |
| 理论视角 | 学生已选的学者/著作/理论 | 从读书报告中提取 |
| 核心机制 | 假设背后的因果链 | 从文献综述的"解释缺口"中提取 |
| 关键概念 | 2-3个核心分析概念 | 从理论框架中提取 |

**输入确认模板**：
```
【已确认的研究要素】
研究问题：[RQ]
选定假设：[H1陈述]
理论视角：[学者/著作/理论]
核心机制：[一句话因果链]
关键概念：[Concept1], [Concept2]

如果以上信息有误，请告知修正后我们再继续。
```

---

### Step 1: 设计数据采集方案（纯网络、低成本）

**核心原则**：数据采集方案必须满足以下四个"可"标准

| 标准 | 要求 | 反面示例 |
|------|------|----------|
| **可理解** | 本科生能在1小时内理解采集逻辑 | 不依赖复杂的API鉴权、逆向工程 |
| **可操作** | 使用基础工具（Excel、Python基础库）即可完成 | 不需要Docker、Kubernetes、GPU服务器 |
| **可负担** | 零成本或接近零成本（<100元） | 不需要购买商业数据库、实地差旅 |
| **可验证** | 采集结果可通过公开渠道交叉验证 | 不依赖无法验证的私有数据 |

**推荐的数据采集工具链**（按技术门槛排序）：

| 工具 | 技术门槛 | 适用场景 | vibe coding辅助 |
|------|----------|----------|-----------------|
| Excel/手动复制 | 无 | 样本量<100，结构化网页 | 不需要 |
| Python + requests/DrissionPage | 低 | 样本量100-500，需自动化 | 强烈推荐 |
| Python + 公开API | 低 | 有免费API可用（如地图API） | 推荐 |
| Wayback Machine / 网页存档 | 无 | 获取历史网页快照 | 不需要 |
| 社交媒体搜索 | 无 | 收集用户讨论、投诉信息 | 不需要 |

**数据采集方案必须包含**：
1. **数据来源**（具体的网站、平台、数据库）
2. **采集字段**（明确列出要采集哪些变量）
3. **采集方法**（手动/自动/半自动）
4. **样本策略**（ purposive / 分层 /  convenience ）
5. **预期样本量**（不需要功效分析，只需合理的数量估计）
6. **时间窗口**（何时采集，为什么选这个时间）
7. **伦理注意**（仅采集公开数据，不涉及个人隐私）

**自然实验锚点的设计**（准实验元素）：

如果研究假设涉及因果推断，必须嵌入一个**自然实验锚点**。常见类型：

| 锚点类型 | 示例 | 本科生可操作性 |
|----------|------|---------------|
| 政策冲击 | 2026年4月35.97亿罚单、2026年6月新规实施 | ⭐⭐⭐⭐⭐ |
| 平台规则变更 | 某平台突然调整排序算法、费率规则 | ⭐⭐⭐⭐ |
| 突发事件 | 某城市突击检查、媒体曝光事件 | ⭐⭐⭐⭐ |
| 地理变异 | 不同城市的监管力度差异（已有公开指标） | ⭐⭐⭐ |
| 时间断点 | 节假日 vs 平日、疫情前后 | ⭐⭐⭐ |

**方案设计检查**：
- [ ] 数据采集是否完全基于网络公开信息？
- [ ] 是否有自然实验锚点？（如果不是因果问题可豁免）
- [ ] 采集工具是否在本科生技术能力范围内？
- [ ] 预期成本是否低于100元？

---

### Step 2: 设计分析方法（基础统计）

**核心原则**：分析方法必须与本科生的统计知识匹配

| 知识层级 | 可用方法 | 不适合的方法 |
|----------|----------|-------------|
| 零基础 | 描述统计、频数表、百分比、交叉表 | 回归、检验 |
| 基础统计（学过概率统计） | 均值比较、t检验、卡方检验、相关分析 | PSM、DID、工具变量 |
| 中级统计（学过计量经济学入门） | 简单OLS回归、固定效应入门 | 面板数据、GMM、非参数 |

**推荐的三层分析结构**（由浅入深，学生可根据能力选择做到哪一层）：

```
第一层：描述性发现（所有学生必须完成）
    - 样本基本特征（频数、均值、标准差）
    - 核心变量的分布（直方图、箱线图）
    - 分组比较（幽灵商户 vs 合规商户的均值差异）

第二层：推断性分析（有统计基础的学生完成）
    - 组间差异检验（t检验 / Mann-Whitney U）
    - 相关分析（Pearson / Spearman）
    - 简单交叉表分析（配合卡方检验）

第三层：多元分析（统计能力较强的学生选做）
    - 简单线性回归（1-2个核心自变量 + 3-5个控制变量）
    - 子样本比较（按城市/品类/平台分组）
```

**分析方案必须包含**：
1. **因变量**如何测量
2. **核心自变量**如何测量
3. **控制变量**有哪些（为什么需要控制）
4. **每步分析的回答什么问题**
5. **预期结果的方向**（支持/不支持假设时的判断标准）

---

### Step 3: 撰写 3.1 [研究内容]

严格按照以下结构撰写，不增不减：

#### 3.1.1 课题研究的主要目标

列出2-4个具体目标，以问题形式呈现（"本研究试图回答……"），或陈述形式（"本研究旨在……"）。

#### 3.1.2 重点难点

| 重点 | 难点 |
|------|------|
| 列出2-3个研究中的关键问题 | 列出2-3个执行中可能遇到的困难 |
| 每个重点附一句话说明为什么重要 | 每个难点附一句话降低难度的方案 |

#### 3.1.3 整体框架（列出提纲或目录）

以**树形目录**或**章节提纲**形式呈现，不是论文正文的展开，而是研究的整体逻辑结构。

```
第一章 [标题]
  1.1 [子标题]
  1.2 [子标题]
第二章 [标题]
  2.1 [子标题]
  ...
```

#### 3.1.4 研究计划及其可行性

包含：
- 数据采集方案（来源、字段、方法、样本量、时间窗口）
- 分析方案（三步分析结构）
- 时间线（以周为单位，6-8周）
- 可行性论证（为什么本科生可以做：技术门槛、成本、时间）
- vibe coding 如何降低技术门槛（如适用）

---

### Step 4: 撰写 3.2 [创新之处]

严格按照以下两个维度撰写：

#### 3.2.1 学术观点创新

- 理论视角的独特性（学生用了什么非默认的理论）
- 核心概念的创新（从理论中提炼出的新概念）
- 与现有文献的对话（现有文献忽略了什么，本研究如何填补）

#### 3.2.2 研究方法创新

- 数据采集方法的独特性（纯网络、低成本、零差旅）
- 自然实验锚点的设计（如何利用公开事件做因果推断）
- vibe coding / AI辅助的工具价值（如适用）
- 方法的可复制性（其他学生能否用同样方法研究其他问题）

---

### Step 5: 保存输出

**文件名**：`literature-to-design-[topic-slug]-[YYYY-MM-DD].md`

**文件内容**：仅包含 3.1 和 3.2 两个部分，不附文献综述原文、不附数据分析结果。

## 质量检查清单

完成输出前，逐一验证：

- [ ] 输出仅包含 3.1 [研究内容] 和 3.2 [创新之处]，无其他章节
- [ ] 3.1 包含：主要目标、重点难点、整体框架（提纲/目录）、研究计划及其可行性
- [ ] 3.2 包含：学术观点创新、研究方法创新

- [ ] 分析方法在本科生统计知识范围内（无PSM、无DID、无IV、无面板模型）
- [ ] 有自然实验锚点（如果是因果问题）
- [ ] 预期成本低于100元
- [ ] 时间线在6-8周内可完成
- [ ] 整体框架列出明确的提纲或目录
- [ ] 可行性论证具体说明"为什么本科生能做"
- [ ] 无数据分析结论、无假设检验结果、无回归表格

## 常见错误与修复

| 错误 | 表现 | 修复 |
|------|------|------|
| 方法过复杂 | 建议PSM、DID、工具变量 | 替换为t检验、简单回归、描述统计 |
| 数据采集成本高 | 需要实地核查、访谈、购买数据 | 替换为纯网络方法：平台抓取+地图验证+名单匹配 |
| 缺少自然实验锚点 | 纯横截面描述，无因果推断意识 | 嵌入公开事件（政策冲击、规则变更）作为时间锚点 |
| 输出格式不符 | 写成完整论文结构（摘要-引言-方法-结果） | 严格按 3.1 + 3.2 格式输出 |
| 包含分析结论 | 写了"预计发现幽灵商户占比30%" | 删除所有预期结果和结论，只写设计方案 |
| 技术门槛过高 | 要求逆向工程、Docker、GPU | 降级为Excel、Python基础库、浏览器自动化 |
